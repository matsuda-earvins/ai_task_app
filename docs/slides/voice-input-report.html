「Web Speech API」とは？
🎙️
声をテキストに変えてくれる
マイクに向かって話すだけで、文字に変換してくれます。スマホの音声入力と同じしくみです。
🌐
ブラウザだけで動く
アプリをインストールしなくても、Chrome などのブラウザがそのまま対応しています。
💰
無料で使える
外部サービスへの課金なしで利用できます。何万回使っても費用はかかりません。
☁️
声はGoogleに送られる
Chrome の場合、音声データは Google のサーバーで処理されてテキストが返ってきます。
処理の流れ
ボタンを押してからタスクが登録されるまでの流れを、2つの方式で比較します。
🎙️ Web Speech API（赤ボタン）
1
マイクボタンをタップ
録音バーが現れ、録音開始。
2
話す
ブラウザがリアルタイムで文字に変換・蓄積。
3
停止ボタンをタップ
蓄積したテキストをそのまま AI に渡す。
4
AI 解析（GPT-5）
タスク名・期限・担当者・優先度を自動抽出。
5
✅ タスク登録
DB に保存してタスク一覧に反映。
🎤 OpenAI Whisper API（青ボタン）
1
マイクボタンをタップ
録音バーが現れ、音声ファイルとして録音開始。
2
話す
音声データを端末内に蓄積（文字変換はまだ）。
3
停止ボタンをタップ
音声ファイルをサーバーに送信。
4
Whisper 文字起こし
OpenAI Whisper API が音声をテキストに変換（1〜3秒）。
5
AI 解析（GPT-5）
タスク名・期限・担当者・優先度を自動抽出。
6
✅ タスク登録
DB に保存してタスク一覧に反映。
違いのポイント：Web Speech API はブラウザがリアルタイムで文字変換するため待ち時間がない。Whisper API は録音停止後にサーバー経由で変換するため1〜3秒の待ち時間が発生するが、固有名詞・英字・口語の精度が高い。

これからの課題
使えるようにはなりましたが、まだ気になっている点があります。
  • Safari・Firefox では使えない Web Speech API は Chrome と Edge にしか対応していません。Safari や Firefox のユーザーが増えてきたら、OpenAI Whisper API（全ブラウザ対応）への切り替えを検討します。
  • 聞き間違えたときの対処 音声認識が間違えると、そのままタスクに登録されてしまいます。「〇〇として登録しました」というお知らせを出して、すぐ編集できるよう誘導する仕組みを検討しています。
  • いろんな話し方での精度確認 専門用語が多い内容・早口・周囲がうるさい環境など、さまざまなケースで正しく認識できるか確認が必要です。
  • アプリを開かずに登録できるようにしたい 将来的には、ブラウザ拡張やブックマークレットとして音声ボタンを独立させ、アプリを開かなくても「話すだけでタスクを登録」できる体験を目指しています。
